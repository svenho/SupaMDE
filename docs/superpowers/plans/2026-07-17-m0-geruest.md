# M0 — Projekt-Gerüst Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ein lauffähiges, modernes Projekt-Gerüst für SupaMDE aufsetzen, sodass `npm run build`, `npm test`, `npm run lint` und `npm run typecheck` grün durchlaufen.

**Architecture:** TypeScript-Library mit Vite im Library-Mode (ESM + UMD Output). Unit-Tests mit Vitest in jsdom-Umgebung. Linting via ESLint (flat config) + typescript-eslint + Prettier. Die öffentliche `SupaMDE`-Klasse existiert in M0 nur als leeres, testbares Skelett — die Editor-Funktionalität kommt erst in M1.

**Tech Stack:** TypeScript 5.9, Vite 8 (Library-Mode), Vitest 4 + jsdom, ESLint 10 (flat config), typescript-eslint 8, Prettier 3.

## Global Constraints

- **TypeScript-Version: `^5.9.3`** — NICHT auf 7.x heben. Grund: `typescript-eslint@^8.64` deklariert Peer-Dependency `typescript >=4.8.4 <6.1.0`; TS 7 ist offiziell nicht unterstützt.
- **Paketname:** `supamde`. **Hauptklasse:** `SupaMDE` (benannter Export + Default-Export).
- **Modul-System:** ESM-first (`"type": "module"` in package.json). Build erzeugt zusätzlich UMD für `<script>`-Einbindung.
- **Node:** Toolchain verlangt `^20.19.0 || ^22.13.0 || >=24` (Ableitung: jsdom 29 & ESLint 10 sind am strengsten; Vite 8 will `^20.19.0 || >=22.12.0`). Umgebung: **v22.18.0** — erfüllt alles. Ein pauschales „Node ≥ 20" wäre falsch (z. B. 20.10 oder 22.5 würden von jsdom/ESLint abgelehnt).
- **Der Ordner `easy-markdown-editor/` bleibt unangetastet** — reine Referenzvorlage, per `.gitignore` ignoriert.
- **TDD:** Jede nicht-triviale Logik wird test-first entwickelt. Häufige, kleine Commits.
- **Sprache:** Code-Kommentare und Doku auf Deutsch (Projekt-Konvention), Identifier auf Englisch.


## Task 6: SupaMDE-Skelett-Klasse (test-first)

**Files:**
- Create: `src/index.ts`
- Create: `src/__tests__/index.test.ts`

**Interfaces:**
- Consumes: `VERSION` aus `src/version.ts` (Task 5).
- Produces:
  - `export interface SupaMDEOptions { element?: HTMLElement | null }`
  - `export class SupaMDE { constructor(options?: SupaMDEOptions); readonly options: SupaMDEOptions; static readonly version: string }`
  - `export default SupaMDE`
  - `export { VERSION } from './version'`
  Dies ist das Skelett, das M1 mit der EditorView-Erzeugung füllt. In M0 speichert der Konstruktor nur die (normalisierten) Optionen.

- [ ] **Step 1: Failing test schreiben**

`src/__tests__/index.test.ts`:

```ts
import { describe, it, expect } from 'vitest';
import SupaMDE, { SupaMDE as NamedSupaMDE, VERSION } from '../index';

describe('SupaMDE (Skelett)', () => {
  it('exportiert dieselbe Klasse als Default- und benannten Export', () => {
    expect(SupaMDE).toBe(NamedSupaMDE);
  });

  it('lässt sich ohne Optionen instanziieren', () => {
    const editor = new SupaMDE();
    expect(editor).toBeInstanceOf(SupaMDE);
    expect(editor.options).toEqual({});
  });

  it('übernimmt übergebene Optionen', () => {
    const el = document.createElement('textarea');
    const editor = new SupaMDE({ element: el });
    expect(editor.options.element).toBe(el);
  });

  it('stellt die Version als statische Eigenschaft bereit', () => {
    expect(SupaMDE.version).toBe(VERSION);
  });
});
```

- [ ] **Step 2: Test ausführen, Fehlschlag verifizieren**

Run: `npx vitest run --environment jsdom src/__tests__/index.test.ts`
Expected: FAIL — Modul `../index` bzw. die Exporte existieren noch nicht. (Flag schon hier, damit derselbe Befehl in Step 4 nach der Implementierung durchläuft.)

- [ ] **Step 3: Minimale Implementierung**

`src/index.ts`:

```ts
import { VERSION } from './version';

/** Konfigurationsoptionen für SupaMDE. Wird in späteren Meilensteinen erweitert. */
export interface SupaMDEOptions {
  /** Das Textarea-Element, an das der Editor gebunden wird. */
  element?: HTMLElement | null;
}

/**
 * SupaMDE — moderner Markdown-Editor auf Basis von CodeMirror 6.
 *
 * In M0 ist dies bewusst nur ein Skelett: Der Konstruktor nimmt Optionen
 * entgegen und legt sie ab. Die Editor-Erzeugung (EditorView) folgt in M1.
 */
export class SupaMDE {
  /** Aktuelle SupaMDE-Version. */
  static readonly version = VERSION;

  /** Die (aktuell unveränderten) Optionen dieser Instanz. */
  readonly options: SupaMDEOptions;

  constructor(options: SupaMDEOptions = {}) {
    this.options = options;
  }
}

export { VERSION } from './version';
export default SupaMDE;
```

- [ ] **Step 4: Test ausführen, Erfolg verifizieren**

Run: `npx vitest run --environment jsdom src/__tests__/index.test.ts`
Expected: PASS (4 Tests grün). Das `--environment jsdom`-Flag ist hier **erforderlich**, weil der Test `document` nutzt und die jsdom-Umgebung erst in Task 7 dauerhaft über `vite.config.ts` gesetzt wird. Ab Task 7 entfällt das Flag (dann via Config).

- [ ] **Step 5: Commit**
